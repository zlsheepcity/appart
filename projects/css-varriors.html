They fight against inline styling

It's about CSS & Markup.

Sometimes we need to add custom value without any hurt to current design system. Inline styling isn't a good solution.

If only we always could add .my-custom-class …
But it's not.

CSS varriors

These are two dozen CSS classes whose names coincide with CSS properties. The custom values are stored in CSS variables, not in property itself.

<div class="width" style="--width: 137px" >
  • The markup remains safe for reuse.
  • The designer remains calm.
  • Memorable logic with CSS property names.
<!-- Typical System Example -->

<blockquote class="padding-sm">...</blockquote>
<blockquote class="padding-md">...</blockquote>

<!-- Avoid Inline Styling -->

<blockquote style="padding-left: 27.5vw">...</blockquote>

<!-- Use Varriors Instead -->

<blockquote class="padding-left" style="--padding-left: 27.5vw">
    <p> Var. Var never changes. </p>
    <cite> Ron Perlman (almost correct quote) </cite>
</blockquote>

<!-- Each Varrior Works Like -->

<style>
    .padding-left {
        padding-left: var(--padding-left);
    }
</style>
